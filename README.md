# PsicoPanel - estructura inicial frontend/backend

## Qué incluye
- `frontend/`: refactor del prototipo visual actual en componentes, screens, utils, data y modals.
- `backend/`: scaffold inicial con controllers, models, routes, modules y config.

## Objetivos de esta versión
- Mantener el estilo visual actual.
- Conservar features futuras como Finanzas e IA, pero en estado bloqueado/greyed out.
- Separar la base del proyecto para que podamos conectar API real después.

## Siguiente paso sugerido
1. Conectar el frontend a un servicio `api/` mock.
2. Diseñar tablas reales de PostgreSQL.
3. Migrar pacientes, notas, tareas y citas al backend.
